Vocabulary measurement

Adaptive vocabulary size testing from a frequency list

A practical summary of how to estimate how many words a learner knows in a target foreign language, starting from a ranked frequency list and improving the test with response data.

The proven baseline

Use Computerized Adaptive Testing, backed by Item Response Theory. For a vocabulary test, the simplest strong model is the Rasch model. Each user has one latent vocabulary ability, and each word has one calibrated difficulty.

P(knows word i | theta, b_i) = logistic(theta - b_i)

Here, theta is the user's vocabulary ability and b_i is the difficulty of item i. A word with difficulty near the current user ability gives the most information. That is why the adaptive test can be short: after each answer it updates theta, then chooses the next word near that new estimate.

Frequency rank is useful, but only as a starting prior. The adaptive test should eventually use calibrated item difficulty from real responses.

A good production item bank mixes yes/no recognition items, multiple-choice meaning checks, and pseudowords. Yes/no items are fast. Multiple-choice items reduce self-report inflation. Pseudowords detect guessing or low attention.

How to calibrate item difficulty

Calibration means collecting enough pilot responses to estimate each word's real difficulty and remove bad items. The test should not start fully adaptive during this phase, because full adaptivity over-samples some words and starves others.

  1. Build an initial item pool from frequency bands. Use lemmas or word families, not raw inflected forms.
  2. Run a balanced pilot. Each user gets words from very common, common, middle, rare, and very rare bands.
  3. Convert answers to binary responses. Real word known or multiple-choice correct is 1. Unknown or wrong is 0.
  4. Use pseudowords and multiple-choice misses to filter unreliable sessions before fitting the model.
  5. Fit a Rasch model to estimate user abilities and item difficulties.
  6. Discard weak items, then refit.

Common discard rules: too few responses, poor item-total correlation, bad Rasch fit, broken distractors, extreme ambiguity, or high false-positive rates. A practical fit cutoff is to inspect infit/outfit mean-square values and remove items far from the expected range, often above about 1.3 for noisy items.

{
  "word": "chien",
  "rank": 814,
  "type": "yes-no",
  "difficulty": -1.22,
  "fit": { "infit": 0.98, "outfit": 1.04 },
  "responses": 427
}

After this point, adaptive testing should select by difficulty, not raw frequency rank.

How to map ability to word count

After Rasch calibration, each real item has both a frequency rank and a calibrated difficulty. Fit a monotone relationship between log(rank) and item difficulty.

difficulty = f(log(rank))

A simple first version is linear regression:

b_i = alpha + beta * log(rank_i)

Then invert it:

rank(theta) = exp((theta - alpha) / beta)

A better version uses monotone spline or isotonic regression. The best scoring method estimates expected known words across the whole vocabulary universe:

V(theta) = sum over ranks r of logistic(theta - f(log(r)))

This is better than a hard boundary because real vocabulary is not contiguous. A learner can know rare cognates and miss common abstract words. The expected sum handles that uncertainty.

How many users are needed

Plan by responses per item, not raw user count.

users_needed = (real_items * target_responses_per_item) / real_items_per_user

Calibration level Target responses per item Example: 250 words, 30 real answers/test
Rough MVP 100 About 834 users
Usable product 250-400 About 2,100-3,400 users
Strong public scoring 500-1000 About 4,200-8,400 users

For a small product, a reasonable path is 500-1000 users for the first pilot, 2000-3000 for decent calibration, and 5000+ before treating the score as robust.

A balanced pilot test could show each user 5 very common words, 8 common words, 8 middle words, 6 rare words, 3 very rare words, and 5-8 pseudowords or check items.

Implementation shape

The adaptive loop is small once the item bank is calibrated.

start posterior over theta
repeat until 30 items or confidence interval is narrow:
  pick unused item with difficulty nearest current theta
  ask yes/no or multiple-choice item
  score answer as 1 or 0
  update posterior over theta
estimate vocab size from final theta
show attention/trust indicator from pseudowords and check items

For fitting the Rasch model, use an established package first. In R, TAM, mirt, or eRm are common choices. In Python, py-irt or a custom logistic optimizer can work, but item-fit diagnostics are easier with mature psychometric tooling.

The Polish Vocabulary Size Test is a useful concrete benchmark. It used Item Response Theory, Computerized Adaptive Testing, yes/no items, multiple-choice checks, and pseudowords. It reports 30 stimuli, about two minutes of test time, conversion from logits to word counts, and response filtering through an attention index.
