Vocabulary measurement

Adaptive vocabulary size testing from a frequency list

Short guide. Beginner version first. Advanced version for implementation details.

Core idea

Question

How many words does a learner probably know?

Best answer

Use an adaptive test. It starts with a rough guess, asks useful words, updates the guess, stops when confident enough.

Adaptive test

Adaptive means the next question depends on previous answers. If easy words are known, ask harder words. If hard words are missed, ask easier words.

Terms

Frequency list

A list of words sorted by commonness. Rank 1 is most common. Rank 20000 is much rarer.

Item

One test question. Usually one word plus yes/no or a meaning choice.

Item difficulty

How hard that word is for real learners. Not always same as frequency. Cognates can be easy. Abstract common words can be hard.

Rasch model

A simple math model. It says: user ability minus word difficulty gives chance the user knows the word.

Calibration

Learning real word difficulty from response data.

How the test works

  1. Pick words from the frequency list.
  2. Ask learners if they know them, or ask meaning questions.
  3. Add fake words. Fake words catch guessing.
  4. Use early answers to estimate level.
  5. Choose next word near that level.
  6. Stop after about 25-40 real words.
  7. Return vocabulary estimate plus uncertainty.

Do not trust raw frequency alone. Use it to start. Use real answers to improve.

Calibration

Why needed

Some words are bad test items. Ambiguous, too easy, too hard, bad translation, common typo, obvious cognate. Calibration finds these.

How to do it

Run a pilot test. Do not make it fully adaptive yet. Give each user a balanced mix of easy, medium, and rare words.

What to keep

Keep words with enough answers, sensible difficulty, and good fit. Drop words that behave strangely.

From answers to word count

After calibration, each word has two numbers: frequency rank and real difficulty.

Fit a curve between them. Then user ability can be turned into estimated word count.

Better scoring does not say "user knows every word up to rank X". Real knowledge is patchy. Better: add up the probability the user knows each word.

How many users

Count responses per word, not just users.

users needed = words * responses per word / real words per test

Goal Responses per word 250 words, 30 answers/test
Rough pilot 100 About 834 users
Decent product 250-400 About 2100-3400 users
Strong scoring 500-1000 About 4200-8400 users

Practical target: 500-1000 users for pilot, 2000-3000 for useful calibration, 5000+ for strong public scoring.

Result to show user

  • Estimated known words.
  • Range, like 7200-8400 words.
  • Trust flag from fake words and checks.
  • Short note: receptive vocabulary, not speaking ability.

PVST, the Polish Vocabulary Size Test, is a useful example. It appears available at myvocab.info/pl.

Model

Use Computerized Adaptive Testing with Item Response Theory. Simple baseline: Rasch.

P(x_i = 1 | theta, b_i) = logistic(theta - b_i)

theta is user ability. b_i is item difficulty. Pick next item by maximum information, usually difficulty nearest current posterior mean or mode.

For multiple choice, score 1 only when the learner claims knowledge and chooses correctly. Pseudowords do not estimate ability. They estimate attention and false-positive behavior.

Item bank

  • Use lemmas or word families, not raw inflected forms.
  • Sample from log-frequency bands.
  • Keep item type: yes/no, meaning choice, pseudoword.
  • Track language-specific issues: cognates, homographs, proper names, register.

Calibration

Do a balanced pilot before full adaptivity. Full adaptivity creates sparse, biased exposure. Each item needs enough response data.

  1. Collect user, item, response, item type, duration.
  2. Filter sessions with high pseudoword false positives, very fast timing, or many claim-known meaning errors.
  3. Fit Rasch with clean binary responses.
  4. Remove bad items: low N, bad item-total correlation, poor infit/outfit, broken distractors, ambiguity.
  5. Refit. Freeze calibrated bank for scoring.

Frequency rank is prior ordering only. After calibration, item selection should use b_i.

Rank to difficulty mapping

Each calibrated item has frequency rank and Rasch difficulty. Fit monotone relationship:

difficulty = f(log(rank))

Simple first pass:

b_i = alpha + beta * log(rank_i)

Then invert:

rank(theta) = exp((theta - alpha) / beta)

Better: monotone spline or isotonic regression. Best scoring:

V(theta) = sum over ranks r of logistic(theta - f(log(r)))

This handles patchy knowledge better than a hard cutoff.

Sample size

Size by responses per item.

users_needed = real_items * target_responses_per_item / real_items_per_user

Calibration level Responses per item 250 items, 30 real answers/test
MVP 100 834 users
Product 250-400 2100-3400 users
Strong 500-1000 4200-8400 users

Adaptive loop

start posterior over theta
repeat:
  choose unused item with max expected information
  ask item
  score x_i as 0 or 1
  update posterior
  stop if item limit hit or vocab CI narrow enough
return expected vocabulary size, CI, attention flag

Use an established calibration package first. R: TAM, mirt, eRm. Python: py-irt or custom optimizer, but diagnostics are weaker.
