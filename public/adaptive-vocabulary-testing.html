Vocabulary measurement

Adaptive vocabulary size testing from a frequency list

Staged method: start with a longer frequency-band test, learn real item difficulty, re-score old sessions, then move to a short adaptive test.

Core idea

Do not start with a short fully adaptive test. Start with a longer test that samples the whole frequency list. Use those answers to learn which words are actually easy or hard. Then shorten the test.

The system improves in stages:

  1. Launch with frequency as a rough difficulty guess.
  2. Ask enough words across easy, middle, and rare bands.
  3. Store every answer.
  4. Calibrate real word difficulty from response data.
  5. Re-score old tests with the better model.
  6. Move to a shorter adaptive test once the word bank is stable.

Frequency rank gets us started. User responses make the test accurate.

Terms

Frequency rank

How common a word is. Rank 1 is most common. Rank 20000 is much rarer.

Item

One test question: a word, usually in a sentence, plus yes/no or a meaning check.

Calibration

Learning real item difficulty from learner answers.

Adaptive test

A test where the next word depends on previous answers. Useful after calibration, risky before calibration.

Quality checks

Fake words, timing, and meaning checks used to detect guessing or low effort.

Rollout

Stage 1: long launch test

Ask about 60-80 real words plus checks. Sample across several frequency bands. This gives a decent first estimate and spreads response data across the item bank.

Stage 2: calibration

Use response data to estimate which words are easier or harder than frequency predicts. Drop bad items: ambiguous words, broken examples, misleading translations, or items that behave strangely.

Stage 3: hybrid test

Use calibrated items for the displayed score. Add a small tail of experimental items to keep improving the bank.

Stage 4: short adaptive test

Once enough items are calibrated, ask mostly adaptive scored items. Keep a few calibration items forever to monitor drift.

Why not adaptive first

A fully adaptive test keeps choosing words near the learner's current level. That is efficient for one learner, but bad for learning the whole bank.

Many words would get almost no answers. Then we could not tell whether those words are good items, bad items, too easy, too hard, or just under-tested.

During calibration, use balanced sampling: easy, middle, rare, and very rare words. Within each band, prefer words with fewer clean responses.

Scoring over time

Before calibration

Estimate the known percentage in each frequency band, correct for fake-word false positives, then multiply by band size.

After calibration

Each word has learned difficulty. The test estimates learner ability, then converts that ability into expected known words.

Old tests improve

Because raw answers are stored, old sessions can be re-scored after calibration. The old result should remain saved, and the revised result should be added with a new model version.

Data to keep

  • User/session id.
  • Item id and item version.
  • Lemma, shown word form, and frequency rank.
  • Answer, meaning-check result, and response time.
  • Whether the item was scored, calibration-only, or quality-only.
  • Test design version and scoring model version.

This is what makes later calibration and re-scoring possible.

How many users

Count clean responses per word, not just users.

users needed = words * responses per word / real words per test

Goal Responses per word 250 words, 30 answers/test
Rough pilot 100 About 834 users
Useful product 250-400 About 2100-3400 users
Strong scoring 500-1000 About 4200-8400 users

Practical target: 500-1000 users for first pilot, 2000-3000 for useful calibration, 5000+ for strong public scoring.

Result to show

  • Estimated receptive vocabulary size.
  • Range, not just one number.
  • Reliability flag from fake words, timing, and meaning checks.
  • Scoring model version.
  • Short note that this measures recognition, not speaking ability.

Method

Use a staged measurement system, not a single fixed test design.

Stage Test shape Score shown
Launch Long stratified test Frequency-band estimate
First calibration Long or semi-balanced test First Rasch estimate, wider CI
Hybrid Adaptive core plus calibration tail Calibrated estimate
Mature Mostly adaptive plus small trickle Short CAT estimate with CI

Full adaptivity is the destination. It is not the launch design.

Item bank

Store item role separately from item type. Role controls whether an item affects the current score.

Role Scored now Purpose
Scored Yes Estimate current user ability.
Calibration No until calibrated Learn future item difficulty.
Quality No Detect false positives, guessing, bots, low effort.

Pilot scoring

Before calibration, use frequency-band sampling and a longer test.

for each band:
  hit_rate = known_real_items / tested_real_items
  false_alarm = fake_words_claimed_known / fake_words_seen
  corrected = (hit_rate - false_alarm) / (1 - false_alarm)
  p_band = clamp(corrected, 0, 1)

vocab_size = sum(band_size * p_band)

Use wide uncertainty. More items reduce sampling error, but only by the square root of item count. Doubling test length helps; it does not make frequency equal true difficulty.

Calibration

Fit a Rasch model once there are enough clean binary responses.

P(x_i = 1 | theta, b_i) = logistic(theta - b_i)

theta is user ability. b_i is item difficulty.

  1. Filter bad sessions: high fake-word false positives, very fast timing, failed meaning checks, incomplete runs.
  2. Convert real-item responses to binary outcomes.
  3. Fit Rasch.
  4. Remove bad items: low N, poor fit, poor item-total relation, broken distractors, ambiguity.
  5. Refit and version the calibrated bank.

Use established calibration tooling first. R: TAM, mirt, eRm. Python: py-irt is possible, but diagnostics are thinner.

Re-scoring

Every calibration round should re-score previous sessions from raw response events.

posterior(theta) proportional to prior(theta) * product(P(response_i | theta, b_i))

Store revised scores as new records, not overwrites. Keep old score, new score, model versions, and confidence intervals.

Hybrid production

Once the first bank is calibrated, split each test into scored items, calibration-tail items, and quality checks.

25-35 calibrated scored items
5-15 calibration items
3-6 quality checks

Only calibrated scored items affect the displayed score. Calibration-tail items improve future versions.

Choose tail items by response scarcity, difficulty uncertainty, weak band coverage, or suspected bad fit. Keep them tolerable: about 40% near level, 30% easier, 30% harder.

Word count

Each calibrated item has frequency rank and Rasch difficulty. Fit a monotone rank-to-difficulty curve:

difficulty = f(log(rank))

Then estimate expected known vocabulary:

V(theta) = sum over ranks r of logistic(theta - f(log(r)))

This handles patchy knowledge better than a hard cutoff rank.

Exit criteria

Size by clean responses per item.

users_needed = real_items * target_responses_per_item / real_items_per_user

Calibration level Responses per item 250 items, 30 real answers/test
Rough pilot 100 834 users
Useful product 250-400 2100-3400 users
Strong public scoring 500-1000 4200-8400 users

Reduce heavy calibration when most production items have 250-400+ clean responses, difficulty SE below about 0.25 logits, acceptable fit, and stable estimates under resampling.

Adaptive loop

start prior over theta
repeat:
  choose unused calibrated item with high expected information
  ask item
  score x_i as 0 or 1
  update posterior over theta
  stop if item limit hit or vocabulary CI is narrow enough
return expected vocabulary size, CI, reliability flags

Mature test shape: 25-35 adaptive scored items, 2-5 calibration trickle items, 2-4 quality checks.
